1. 開催予定を更新する（最も頻度が高い作業）
================================================================

「events.js」を開いて EVENTS の配列を編集します。
サイト全体の「次回開催」情報がここを見て自動更新されます。

------------------------------------------------------------
■ 日程が決まったとき
------------------------------------------------------------

tbd: true  → false に変更
date に日付を 'YYYY-MM-DD' 形式で設定
dayOfWeek に曜日を設定

変更前:
  tbd:      true,
  date:     null,
  dayOfWeek: null,

変更後（例: 6月15日 日曜日）:
  tbd:      false,
  date:     '2026-06-15',
  dayOfWeek: '日',

------------------------------------------------------------
■ connpass のURLが決まったとき
------------------------------------------------------------

connpassUrl の null を実際のURLに変更します。

変更前:
  connpassUrl: null,

変更後:
  connpassUrl: 'https://connpass.com/event/1234567/',

URLを設定すると、各ページの「申込受付前」ボタンが
自動的に「connpassで申込む」ボタンに切り替わります。

------------------------------------------------------------
■ 次の回を追加するとき
------------------------------------------------------------

EVENTS 配列の先頭（コメント行の直下）に新しい項目を追加します。
前回の項目は残しておいても問題ありません。

追加例:
  {
    id:          25,
    tbd:         true,
    date:        null,
    dayOfWeek:   null,
    startTime:   '13:00',
    endTime:     '16:00',
    seasonEmoji: '🌿',   ← 季節に合わせて変更（下記参照）
    monthLabel:  '次回',
    venue:       '小牧市東部の公共施設（予定）',
    connpassUrl: null,
    notes:       '',
  },

seasonEmoji の目安:
  春（3〜5月）: '🌸'
  夏（6〜8月）: '🌿'
  秋（9〜11月）: '🍂'
  冬（12〜2月）: '❄️'

2. お知らせ記事を追加する
================================================================

「news.html」を開き、<div class="news-list"> の中に
記事ブロックを追加します。新しい記事ほど上に書きます。

記事ブロックのテンプレート:

  <article class="news-card fade-in" data-type="タグ種別">
    <div class="news-date">
      <div class="month">月の英語3文字</div>
      <div class="day">日付</div>
      <div class="year">西暦</div>
    </div>
    <div class="news-body">
      <span class="news-tag タグCSSクラス">タグ名</span>
      <h3>記事タイトル</h3>
      <p>記事本文</p>
    </div>
  </article>

タグの種類と設定値:
  開催情報 → data-type="event"  クラス: tag-event
  活動報告 → data-type="report" クラス: tag-report
  お知らせ → data-type="info"   クラス: tag-info

記入例:
  <article class="news-card fade-in" data-type="report">
    <div class="news-date">
      <div class="month">May</div>
      <div class="day">18</div>
      <div class="year">2026</div>
    </div>
    <div class="news-body">
      <span class="news-tag tag-report">活動報告</span>
      <h3>【5月活動報告】〇〇しました</h3>
      <p>本文をここに書きます。</p>
    </div>
  </article>

また、ページ上部の「ピン留め（次回開催）」バナーは
events.js の内容が自動反映されるため編集不要です。

注意: お知らせ記事の本文は多言語化されていません。
日本語で記述するか、本文内に必要な言語を直接記載してください。

3. ページ内のテキストを変更する（多言語対応）
================================================================

ナビゲーション・見出し・説明文など、ページ上の主なテキストは
「i18n.js」で一元管理されています。

テキストを変更するときは i18n.js を開き、
該当するキーの文字列を書き換えます。

対応言語:
  ja （日本語）
  en （英語）
  pt （ポルトガル語）
  vi （ベトナム語）

------------------------------------------------------------
■ テキストの構造
------------------------------------------------------------

const T = {
  ja: {
    'キー名': '日本語テキスト',
    ...
  },
  en: {
    'キー名': 'English text',
    ...
  },
  pt: {
    'キー名': 'Texto em português',
    ...
  },
  vi: {
    'キー名': 'Văn bản tiếng Việt',
    ...
  },
};

同じキー名が各言語セクションに存在します。
すべての言語の同じキーを一緒に更新してください。

------------------------------------------------------------
■ 変更例（ヒーローエリアのキャッチコピー）
------------------------------------------------------------

変更前（ja セクション）:
  'hero-h1': 'コードで未来を<br><span>つくろう！</span>',

変更後:
  'hero-h1': 'コードで世界を<br><span>変えよう！</span>',

同様に en / pt / vi の 'hero-h1' も変更する。

------------------------------------------------------------
■ 新しいテキストブロックを HTML に追加するとき
------------------------------------------------------------

HTML 要素に data-i18n 属性（またはHTMLタグを含む場合は
data-i18n-html 属性）を付けてキー名を指定します。

  <p data-i18n="my-new-key">日本語のデフォルトテキスト</p>

その後、i18n.js の ja / en / pt / vi の全セクションに
同じキー名のエントリを追加します。

  ※ HTML を変更したときに i18n.js の更新を忘れると、
    外国語表示で古い内容が残ったままになります。
    Claude Code を使用している場合は後述の自動チェックが
    機能しますが、手動の場合は必ず一緒に更新してください。

------------------------------------------------------------
■ HTML ファイル内の固定テキスト
------------------------------------------------------------

以下の内容は i18n.js ではなく HTML ファイルに直接書かれています:
  news.html  … 個別の記事本文
  venue.html … 住所・地図の埋め込み

これらは各 HTML ファイルを直接編集してください。

4. 開催場所の情報を更新する
================================================================

「venue.html」を開いて該当箇所を直接書き換えます。

住所の記載箇所（現在「住所確認中」と表示されている部分）:
  <!-- TODO: 正確な住所・郵便番号を確認の上ご記入ください -->
  <dd>愛知県小牧市（住所確認中）</dd>

  ↓ 確認できたら書き換え
  <dd>〒485-XXXX<br>愛知県小牧市〇〇〇〇番地</dd>

Google マップの埋め込み URL の差し替え方:
  1. Google マップ（maps.google.com）で会場を検索
  2. 「共有」→「地図を埋め込む」をクリック
  3. 表示された <iframe> の src="..." の値をコピー
  4. venue.html の iframe タグの src を差し替え

5. お問い合わせフォームのメール送信を有効にする
================================================================

現在はメール送信が未設定です。以下の手順で有効化します。

1. https://formspree.io にアクセス
2. advancation@gmail.com でアカウント作成
3. 「+ New Form」で新しいフォームを作成
4. 表示されたエンドポイント URL を取得
   （例: https://formspree.io/f/xabcdefg）
5. 「script.js」の以下の行を書き換え

  変更前:
    const FORM_ENDPOINT = 'https://formspree.io/f/YOUR_FORM_ID';

  変更後:
    const FORM_ENDPOINT = 'https://formspree.io/f/xabcdefg';

無料プランでは月50件まで受信できます。

6. 季節の演出について（自動切替）
================================================================

トップページのヒーロー（大きなビジュアル）は
毎月の日付を見て自動的に季節の見た目に変わります。
手動での操作は不要です。

  春（3〜5月）: 桃の花・ピンク
  夏（6〜8月）: 新緑・青空
  秋（9〜11月）: 紅葉・給水塔
  冬（12〜2月）: 雪・青紺

手動でテストしたい場合は script.js の以下の行を一時的に
書き換えることで強制的に季節を指定できます:

  const SEASON = getSeason();
  ↓
  const SEASON = 'summer';   // 'spring' / 'summer' / 'autumn' / 'winter'

確認が終わったら元の行に戻してください。

7. ファイルの公開（アップロード）について
================================================================

ファイルを編集したあとは、ホスティングサービスに
アップロードする必要があります。

対象フォルダ: coderdojo/ フォルダ内のすべてのファイル

更新したファイルだけアップロードすれば反映されます。
（events.js だけ更新した場合は events.js だけでOK）
（テキストを変更した場合は i18n.js と変更した HTML の両方をアップロード）

アップロード不要なファイル（ローカル作業専用）:
  scripts/        … 整合性チェックスクリプト（Claude Code 用）
  .claude/        … Claude Code プロジェクト設定
  運用方法.txt    … この文書

8. 多言語整合性チェック（Claude Code 使用時の自動機能）
================================================================

このプロジェクトには、HTML と i18n.js の整合性を自動チェックする
仕組みが組み込まれています。

------------------------------------------------------------
■ 仕組み
------------------------------------------------------------

Claude Code で .html ファイルまたは i18n.js を編集すると、
編集直後に自動でチェックが走り、問題があれば Claude に通知されます。

チェック内容:
  1. HTML で使われているキー（data-i18n / data-i18n-html 属性の値）が
     i18n.js の ja / en / pt / vi 全セクションに存在するか
  2. HTML の日本語デフォルトテキストと i18n.js の ja 値が一致しているか

------------------------------------------------------------
■ 手動でチェックを実行する
------------------------------------------------------------

ターミナルで以下を実行します:

  python3 scripts/check-i18n.py

問題がなければ:
  i18n check: OK — all keys present and ja defaults match

問題があれば:
  === i18n check: issues found ===
  MISSING  [en] 'キー名'  (used in index.html)
  MISMATCH 'キー名'
    HTML default : ...
    i18n.js [ja] : ...

------------------------------------------------------------
■ 新しいキーを追加するときの手順（まとめ）
------------------------------------------------------------

  1. HTML に data-i18n="new-key" を追加
  2. i18n.js の ja セクションに 'new-key': '日本語テキスト' を追加
  3. i18n.js の en セクションに 'new-key': 'English text' を追加
  4. i18n.js の pt セクションに 'new-key': 'Texto em português' を追加
  5. i18n.js の vi セクションに 'new-key': 'Văn bản tiếng Việt' を追加
  6. python3 scripts/check-i18n.py でエラーなしを確認


================================================================
9. 困ったときの確認ポイント
================================================================

ページが真っ白になった
  → HTML ファイルに記述ミス（タグの閉じ忘れなど）がある可能性
  → 直前の変更を元に戻してみる

次回開催情報が更新されない
  → events.js の保存・アップロードを確認
  → tbd: false になっているか確認
  → ブラウザのキャッシュをクリア（Ctrl+Shift+R）

テキストが変わらない
  → i18n.js の保存・アップロードを確認
  → ブラウザのキャッシュをクリア（Ctrl+Shift+R）
  → HTML の data-i18n 属性のキー名と i18n.js のキー名が一致しているか確認
  → python3 scripts/check-i18n.py を実行して整合性を確認

外国語が日本語のままになっている
  → i18n.js の該当言語セクションにキーが存在するか確認
  → python3 scripts/check-i18n.py を実行して MISSING エラーを確認

地図が表示されない
  → venue.html の iframe の src URL を確認
  → Google マップの「共有」から正しい埋め込み URL を取得して差し替え

メールが届かない
  → script.js の FORM_ENDPOINT が正しいか確認
  → Formspree のダッシュボードでフォームが有効か確認
  → 迷惑メールフォルダを確認
